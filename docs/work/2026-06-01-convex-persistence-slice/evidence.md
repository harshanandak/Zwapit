## Fallback / Loading Behavior Notes

- All Convex paths are gated by `isConvexConfigured()` (`VITE_CONVEX_URL`). When
  unset (CI/build/local-without-Convex), every adapter function returns the
  existing local mock data; `convex/browser` is loaded only via dynamic import
  when a URL is present, so SSG/build/tests never evaluate it.
- Astro pages remain server-rendered from `createMockFixture()`, so the static
  HTML (and every acceptance needle) is byte-stable and unchanged.
- The timeline island first-paints from local/seeded state (no blank/loading
  product state), then hydrates from Convex when configured. The "Advance (demo)"
  control persists transitions in Convex when configured; reload re-reads them.
- Convex errors fall back to local behavior (best-effort persistence), so the UI
  never regresses if a deployment is unreachable.

## Known Scope-Guard Conflict (for Codex / user decision)

`scripts/verify-first-visible-slice.mjs::verifyNoScopeDrift` is the prior
first-visible-slice guard. It (a) bans any dependency/import matching
`/clerk|razorpay|convex|stripe|supabase|neon|postgres/i` and (b) restricts branch
changes to a first-slice path allowlist that excludes `convex/`, `src/lib/convex/`,
`docs/work/`, and `.beads/`.

- This test was ALREADY failing on the planning branch before any code change
  (it flagged the `docs/work/**` and `.beads/team-map.jsonl` planning files).
- It now also flags the **authorized** Convex addition for this slice:
  `real integration dependency present: convex`, and convex imports in
  `src/lib/convex/client.ts`, `src/lib/convex/functionRefs.ts`,
  `src/lib/flow/timelinePanel.client.ts`.

Per repo ownership, acceptance verification / hard-exclusion gating is a Codex
task (Tasks 7-9). Claude did NOT modify this guard. Recommended Codex/user action:
update the guard so `convex` is permitted (it is this slice's intended backend)
while still banning the real hard exclusions (clerk/razorpay/stripe/payments/
payout/admin), and add `convex/`, `src/lib/convex/`, `docs/work/`, `.beads/` to the
allowed paths.
