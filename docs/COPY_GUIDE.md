# Copy Guide

Source: ChatGPT shared conversation, extracted 2026-05-28.
Status: Baseline planning decision, not yet implemented.

## Tone

- Friendly.
- Clear.
- Non-corporate.
- Reassuring without legal-heavy language.
- Progressive disclosure: show only what the user needs at that moment.
- Mode-specific: buyer instructions stay in buyer screens, seller instructions stay in seller screens.

## Use

- Official Transfer
- Protected Handoff
- Verify & Redeem
- Waitlist Only
- Cannot List
- Protected payment
- Payout
- Report issue
- Upload to Sell
- Buy with Protection
- Saved
- My Tickets
- Orders
- Transfer needed
- Payout waiting
- Under Review
 
Use "My Tickets" for buyer-side purchases.
Use "Orders" for seller-side received purchases inside Sell.
Avoid using "Sales" as the primary seller-side label.
Do not use "Transactions" in bottom navigation.

## Avoid In UI

- escrow
- settlement
- dispute
- merchant
- fulfilment
- entitlement
- KYC
- linked account
- AMBER
- settlement hold
- payout release job

## Example Translation

Avoid:
- This order is AMBER, settlement hold is active, payout will be released after fulfilment_end_at plus dispute_window.

Use:
- Protected Handoff. The buyer's payment is protected until the trip is completed.
